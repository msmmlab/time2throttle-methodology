# time2throttle-methodology

Public repository for the [Time2Throttle](https://time2throttle.com) test methodology — the measurement standard used to benchmark thermal throttling, surface temperature, and real battery runtime in smart glasses and wearables.

The methodology is published here **before any results are produced**, by design. Reproducibility is the product.

## How to suggest a change

**Option 1 — Open an Issue** (no git required)
Use the [Methodology critique](https://github.com/msmmlab/time2throttle-methodology/issues/new/choose) template. Describe the gap, the section it affects, and your proposed fix. Substantive corrections are credited in the changelog.

**Option 2 — Open a Pull Request**
Fork this repository, make your proposed edit to the relevant `.mdx` file on a branch, and open a PR against `main`. The diff will be publicly visible and reviewed before any merge.

---

## Versioning rules

- The methodology changes only by public versioned revision with a changelog entry — never retroactively.
- Every merged change to a `.mdx` document produces a new version file (e.g. `v0.3.mdx`).
- Prior versions are kept in this repository permanently.
- All changes to `main` require a pull request — direct pushes are blocked.
